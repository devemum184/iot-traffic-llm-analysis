# Технический отчет: Использование LLM для генерации программ анализа сетевого трафика IoT

## 1. Введение
Целью данной работы является исследование возможностей больших языковых моделей (LLM) в автоматизации процессов создания программного обеспечения для анализа сетевого трафика. В рамках задачи было проведено сравнение эффективности моделей машинного обучения, сгенерированных посредством специализированных подсказок (промптов) для LLM, с базовой моделью, написанной аналитиком вручную. 

Исследование проводилось на наборе данных `subhajournal/iotintrusion`, содержащем записи как легитимного (BenignTraffic), так и вредоносного трафика (различные векторы DDoS, DoS, MITM, Recon).

## 2. Методология тестирования
Для проведения объективного сравнения были подготовлены три программных пайплайна:
1. **Baseline Model (Ручная разработка):** Использован алгоритм `RandomForestClassifier`.
2. **LLM Model 1 (Сгенерирована алгоритмом 1):** Использован алгоритм `HistGradientBoostingClassifier`, предложенный LLM как оптимальный для объемных табличных данных.
3. **LLM Model 2 (Сгенерирована алгоритмом 2):** Использован алгоритм `MLPClassifier` (многослойный персептрон).

Для чистоты эксперимента все три модели использовали идентичный механизм загрузки данных (через `kagglehub`) и одинаковый конвейер предобработки (Label Encoding категориальных признаков, StandardScaler, разбиение 80/20).

## 3. Результаты эксперимента

В ходе тестирования на отложенной выборке объемом 209 715 записей были получены следующие метрики качества:

| Модель | Алгоритм | Accuracy | Macro F1-score | Weighted F1-score | Особенности поведения |
| :--- | :--- | :--- | :--- | :--- | :--- |
| **Baseline** | Random Forest | 0.9930 | 0.73 | 0.99 | Наилучший показатель распознавания классов меньшинства. |
| **LLM Model 1** | HistGradientBoosting | 0.9657 | 0.57 | 0.97 | Склонность к ошибкам на слабо представленных классах атак. |
| **LLM Model 2** | MLPClassifier (NN) | 0.9849 | 0.67 | 0.98 | Высокая точность классификации, сопоставимая с базовой моделью. |

### Анализ по классам атак
Все три модели столкнулись с проблемой дисбаланса классов, характерной для датасетов сетевого трафика. 
Атаки с малым количеством примеров (например, `Backdoor_Malware`, `SqlInjection`, `XSS`, `Uploading_Attack` — от 3 до 33 примеров в тестовой выборке) показали метрики Precision и Recall, равные 0.0 у всех трех моделей. 

В то же время массивные DDoS-атаки (`DDoS-ICMP_Flood`, `DDoS-UDP_Flood`, `DDoS-TCP_Flood` и др.) выявлялись стабильно всеми инструментами с точностью близкой к 1.00. Базовая ручная модель `Random Forest` продемонстрировала лучшую обобщающую способность (Macro F1 = 0.73), эффективнее справляясь с умеренно редкими классами по сравнению с предложенными LLM решениями.

## 4. Выводы
1. **Эффективность генерации:** Эксперимент подтверждает, что LLM способны генерировать синтаксически корректные и логически верные пайплайны машинного обучения для классификации сетевого трафика "из коробки". Основной массив сетевых аномалий выявляется сгенерированным кодом стабильно.
2. **Сравнение подходов:** Классический `Random Forest`, выбранный вручную, превзошел по метрикам градиентный бустинг и нейронную сеть, предложенные языковыми моделями без дополнительного гиперпараметрического тюнинга. LLM отдают предпочтение более сложным алгоритмам, которые могут требовать более тонкой настройки под конкретный распределенный датасет.
3. **Ограничения LLM:** Несмотря на высокую базовую `Accuracy` (от 96.5% до 98.4%), сгенерированные скрипты без явного указания в промпте игнорируют проблему жесткого дисбаланса классов (imbalanced data), что приводит к пропуску критичных, но редких уязвимостей. Для применения в реальных ИБ-системах сгенерированный код требует доработки аналитиком (например, добавления техник SMOTE или взвешивания классов).
